Astromech Operating System

A turnkey animation scripting and control solution for R2 builders. AstrOs provides a complete ecosystem so you can focus on bringing your astromech to life.

View on GitHub

Projects

The AstrOs ecosystem is made up of several interconnected repositories.

AstrOs.Server

TypeScript

A Node.js/Express server with a Vue 3 frontend for designing animations and managing microcontroller configurations. Runs in a Docker container on a single board computer (Raspberry Pi, Orange Pi) with serial port access.

  • ✦ Animation scripting interface
  • ✦ Microcontroller configuration management
  • ✦ Playlist and remote control support
  • ✦ Docker-based deployment
View Repository

AstrOs.ESP

C / ESP32

The firmware for ESP32-based AstrOs microcontroller nodes. Receives animation commands from the AstrOs Server via serial communication and drives servos, sound boards, and other droid electronics.

  • ✦ ESP32 firmware via PlatformIO
  • ✦ Serial communication protocol
  • ✦ Servo, sound, and I2C module control
  • ✦ Pairs with AstrOs.Server
View Repository

AstrOs.Screen

C / ESP-IDF

A portable touchscreen remote for triggering AstrOs animations. Runs on an Elecrow 5" ESP32 display, syncs button configurations from the AstrOs Server, and allows on-the-floor control of your droid at events.

  • ✦ Touch-friendly animation trigger grid
  • ✦ WiFi connectivity with panic stop
  • ✦ Syncs config from AstrOs Server
  • ✦ Built with LVGL & SquareLine Studio
View Repository

How It Works

AstrOs connects a scripting server to one or more ESP32 nodes in your droid, with optional touchscreen and e-paper peripherals for event use.
